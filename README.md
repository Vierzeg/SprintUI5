Описание проекта

Этот проект содержит автоматические тесты для веб-сайта Stellar Burgers, который представляет
собой интернет-магазин бургеров. Тесты охватывают различные аспекты функциональности сайта,
включая регистрацию, вход в личный кабинет, работу с конструктором бургеров и проверку
функционала профиля пользователя.

Тесты написаны с использованием Selenium для автоматизации браузерных действий и pytest для
удобного тестирования и управления тестами.


Структура проекта

Проект включает в себя следующие файлы и директории:

tests/fixtures.py: фикстуры для тестирования (инициализация драйвера, генерация данных для регистрации и авторизации).

tests/locators.py: локаторы для элементов на страницах сайта.

tests/test_account.py: тесты для проверки функционала личного кабинета (переход в ЛК, выход, авторизация).

tests/test_constructor.py: тесты для проверки работы конструктора бургеров.

tests/test_registration.py: тесты для проверки функционала регистрации и обработки ошибок.


Требования

Для запуска тестов требуется:

Python 3.7 или выше

Selenium

pytest

WebDriver (например, ChromeDriver)

Убедитесь, что у вас установлен браузер Google Chrome и его соответствующий WebDriver.


Установка зависимостей

1. Клонируйте репозиторий:
git clone https://github.com/your-repository.git
cd your-repository

2. Создайте и активируйте виртуальное окружение:
python -m venv venv
source venv/bin/activate  # Для Linux/macOS
venv\Scripts\activate  # Для Windows

3. Установите необходимые зависимости:
pip install -r requirements.txt

Если requirements.txt отсутствует, установите зависимости вручную:
pip install selenium pytest


Запуск тестов

Чтобы запустить тесты, используйте команду:
pytest

По умолчанию pytest выполнит все тесты в проекте. Если вам нужно запустить только определённые тесты, используйте параметр -k:
pytest -k "test_name"

Описание тестов
Тесты для личного кабинета
test_profile_access — Проверяет возможность перехода в личный кабинет.


test_const_from_profile — Проверяет переход в конструктор из личного кабинета.


test_logo_from_profile — Проверяет отображение логотипа в личном кабинете.


test_login_from_main_button — Проверяет вход в личный кабинет с главной страницы.


test_login_profile_access — Проверяет авторизацию через кнопку "Личный кабинет".


test_login_reg_form — Проверяет авторизацию через окно регистрации.


test_login_rec_pass — Проверяет возможность входа через страницу восстановления пароля.


test_out_from_profile — Проверяет выход из личного кабинета.


Тесты для конструктора бургеров
test_sauces_section — Проверяет доступность секции "Соусы" в конструкторе.


test_buns_section — Проверяет доступность секции "Булки" в конструкторе.


test_fillings_section — Проверяет доступность секции "Начинки" в конструкторе.


Тесты для регистрации
test_successful_registration — Проверяет успешную регистрацию с правильными данными.


test_incor_pass_registration — Проверяет регистрацию с неправильным паролем и вывод ошибки.

Структура фикстур
driver: Фикстура для инициализации ChromeDriver для тестов с глобальной областью видимости (сессия).


driver_acc: Фикстура для инициализации ChromeDriver для тестов с функциональной областью видимости (вход в личный кабинет).


driver_reg: Фикстура для инициализации ChromeDriver для тестов с функциональной областью видимости (регистрация).


driver_log: Фикстура для инициализации ChromeDriver для тестов с функциональной областью видимости (вход).


generate_email: Генерирует уникальный email с случайным русским именем.


generate_password: Генерирует случайный пароль.


registration_data: Объединяет email и пароль для использования в тестах.


Локаторы
Локаторы для элементов на страницах сайта хранятся в файле locators.py. Они используются для поиска нужных элементов при тестировании.
Пример локаторов:
REGISTER_NAME_FIELD: Поле для имени при регистрации.


REGISTER_EMAIL_FIELD: Поле для email при регистрации.


REGISTER_PASSWORD_FIELD: Поле для пароля при регистрации.


REGISTER_BUTTON: Кнопка регистрации.


LOGIN_EMAIL_FIELD: Поле для email при входе.


LOGIN_PASSWORD_FIELD: Поле для пароля при входе.


PROFILE_BUTTON: Кнопка перехода в личный кабинет.


CONSTRUCTOR_BUTTON: Кнопка перехода в конструктор.


Дополнительная информация
Тесты используют фикстуры для генерации данных (например, email и пароль).


Все тесты используют явные ожидания с помощью WebDriverWait, чтобы гарантировать корректную загрузку страниц и элементов.


Тесты используют случайные данные для регистрации, чтобы обеспечить уникальность каждой сессии.
